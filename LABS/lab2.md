# Пояснення, що станеться, якщо видалити атрибут key (перевірте в консолі розробника)

 Видалення key у методі students.map викличе попередження в консолі про відсутність унікального ідентифікатора для елементів списку. Це завадить React ефективно оновлювати окремі рядки, що призведе до помилок у відображенні даних при зміні масиву.

## Котрольні запитання

1. Навіщо потрібен проп key у списках React? (Відповідь пов&#39;язана з ідентифікацією елементів при Reconciliation).

 Він допомагає React під час Reconciliation точно ідентифікувати, які елементи списку були змінені, додані або видалені, щоб оновлювати лише необхідні частини DOM.

2. Чому не рекомендується використовувати індекс масиву (index) як ключ? (Це може призвести до помилок при зміні порядку елементів або їх видаленні).

  Використання індексу як ключа спричиняє помилки у стані компонентів (наприклад, в інпутах) при сортуванні, видаленні або додаванні елементів у початок списку.

3. У чому різниця між імперативним створенням списку (цикл for) та декларативним (map)?

  Цикл for описує процес покрокового створення списку (як робити), тоді як map описує результат — трансформацію масиву даних у масив елементів (що отримати).

4. Як передати всі властивості об&#39;єкта в компонент одним рядком? (Spread attributes: {...item}).
  Це робиться за допомогою spread-оператора: <Component {...item} />.
