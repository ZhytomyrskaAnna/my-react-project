## Контрольні запитання

**1. Чому при рендерингу масивів у React обов&#39;язково використовувати атрибут key? Що станеться, якщо його не вказати? (Див. Лекцію 2).**

Атрибут key дозволяє React ідентифікувати елементи при Reconciliation (порівнянні віртуального DOM), щоб оновлювати лише змінені вузли, а не весь список. Без нього в консолі з'явиться попередження, а продуктивність та коректність відображення стану (наприклад, введеного тексту в інпутах) погіршаться.
   
**2. У чому різниця між методами .map() та .forEach()? Чому .forEach() не підходить для відображення елементів безпосередньо в JSX?**

Метод .map() створює та повертає новий масив із результатами виклику функції, що дозволяє React рендерити його прямо в JSX. Метод .forEach() лише перебирає елементи й повертає undefined, тому він не може нічого відобразити в дереві компонентів.
   
**3. Чому не рекомендується використовувати індекс масиву (index) як ключ,якщо список може змінюватися (сортуватися, фільтруватися)?.**

При зміні порядку, фільтрації або видаленні елементів індекси перераховуються, що змушує React прив'язувати стан (state) до позиції в масиві, а не до конкретного об'єкта. Це призводить до візуальних багів, коли дані з одного рядка "перестрибують" у інший.
