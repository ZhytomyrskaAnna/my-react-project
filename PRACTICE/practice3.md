# Практична робота №3

## 1. Порожній стан (Empty State)
Реалізовано за допомогою тернарного оператора, який перевіряє довжину відфільтрованого масиву studentsToDisplay:

```tsx
{studentsToDisplay.length === 0 ? (
  <p style={{ textAlign: 'center', color: 'gray' }}>За вашим запитом нікого не знайдено 🔎</p>
) : (
  <ul>{/* Рендеринг списку */}</ul>
)}
```

## 2. Динамічні класи
Використано тернарний оператор для порівняння активного таба (activeTab) із назвою кнопки. Це додає клас styles.activeTab та змінює variant кнопки:

```tsx
<Button 
  onClick={() => setActiveTab('list')} 
  className={activeTab === 'list' ? styles.activeTab : ''}
  variant={activeTab === 'list' ? 'primary' : 'secondary'}
>
  Всі студенти
</Button>
```

## 3. Захист від помилок (Nullish Coalescing)
Використано оператор нульового злиття (??), який виводить альтернативний текст, якщо значення score дорівнює null або undefined:

```tsx
{studentsToDisplay.map((student) => (
  <li key={student.id}>
    <strong>{student.name}</strong> — 
    Бал: {student.score ?? "Оцінка відсутня"}
  </li>
))}
```

## 4. Відповіді на контрольні запитання
Чому if/else не можна в JSX? Оскільки JSX — це вираз, а if/else — інструкція. React очікує всередині {} результат, який можна відрендерити.
Небезпека 0 в &&: Якщо ліва частина умови дорівнює 0, React виведе цифру "0" на екран, замість того щоб нічого не показувати.
Складна логіка (5+ варіантів): Найкраще використовувати об'єктне відображення (Object Map) або винести switch у окрему функцію.
Тернарний оператор vs &&: Тернарний використовуємо, коли є альтернатива (if-else), а && — коли компонент або є, або його взагалі немає.
Механізм перерендерингу: React реагує на зміну стану через useState. При виклику set функції він порівнює значення і оновлює потрібну частину DOM.
